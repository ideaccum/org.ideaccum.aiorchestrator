# RULE_MEMORY.md

コンテキスト圧迫、コンテキスト溢れ、他セッションとの記憶共有に @.orchestrator/memory/MEMORY.md に保存する。  


## メモリファイル構成

- @.orchestrator/memory/MEMORY.md に追記、更新する（ファイルが存在しなければディレクトリを含めて作成する）。  
- @.orchestrator/memory/MEMORY.md 自体も肥大化することを防止するため、@.orchestrator/memory/MEMORY.md は目次の位置づけとして、カテゴリごとに @.orchestrator/memory/MEMORY_*.md に分割して管理する。  


## メモリ記録内容

- 他セッションや他エージェントへ共有すべき情報を記録する。  
- 他セッションや他エージェントが類似のタスクを実行する際に不要に冗長的な作業をしたり、不要なトークン消費を行わないようにすることが目的。  


## メモリ書き込みタイミング

- 自分自身の発言ターン終了後に**必ず**書き込む（ファイルが存在しなければディレクトリを含めて作成する）。  
- 今までのメモリ内容から状況が変わっていない場合は書き込みは不要。  
- すべてのエージェントが書き込みを行う。  


## エージェントメモリ利用

- 会話の邪魔になるので、**エージェントは、メモリを確認、記録、更新する行動やメモリファイルを作成する行動を発言することは絶対に禁止**。  
- 自身が知らない情報が必要な場合、まず @.orchestrator/memory/MEMORY.md に情報が存在しないか確認する。  


## リーダーエージェントターン終了時（必須）

- **リーダーエージェント発言ターン終了時にメモリ更新が全く行われないないことは許可しない（必ず書き込みを行う）**。
